### Task 2: Label prefix highlight + target emphasis

**Files:**
- Modify: `src/lib/game/render/enemy-renderer.ts`

Label texture draw becomes two-segment: typed prefix in accent (`#facc15`), remainder
in white; measure prefix via `ctx.measureText` after setting the same font, draw both
runs manually (drop `drawText`, use raw context + `texture.update()`):

```ts
function drawLabel(v: EnemyVisual, word: string, typedCount: number, isTarget: boolean) {
	const text = `${word}:${typedCount}:${isTarget ? 1 : 0}`;
	if (text === v.lastText) return;
	v.lastText = text;
	const ctx = v.texture.getContext();
	ctx.clearRect(0, 0, 256, 64);
	const font = isTarget ? "bold 44px monospace" : "bold 36px monospace";
	// biome-ignore lint/suspicious/noExplicitAny: ICanvasRenderingContext lacks font metrics typing
	const c = ctx as any;
	c.font = font;
	const typed = word.slice(0, typedCount);
	const rest = word.slice(typedCount);
	const typedW = c.measureText(typed).width;
	const totalW = typedW + c.measureText(rest).width;
	const x = (256 - totalW) / 2;
	const y = 44;
	c.fillStyle = "#facc15";
	c.fillText(typed, x, y);
	c.fillStyle = isTarget ? "#ffffff" : "#9ca3af";
	c.fillText(rest, x + typedW, y);
	v.texture.update();
}
```

`sync()` passes `state.targetId === e.id`. Target's body gets an emissive boost
(store base emissive on the visual, multiply when targeted). Snapshot regen rides.

Commit: `feat(game): highlight typed prefix and locked target on labels`

### Task 3: Effects — deaths, hits, shake, status tints

**Files:**
- Create: `src/lib/game/render/effects.ts`
- Modify: `src/lib/game/render/loop.ts`, `src/lib/game/render/scene.ts`, `src/lib/game/render/enemy-renderer.ts`

Sim exposes no event log (by design); derive events by diffing consecutive states in
the render layer (loop already holds prev/next): enemies present before and absent now
→ death burst at last position (kill vs leak distinguished by `playerHp` delta);
`playerHp` drop → screen shake (camera target jitter decaying over ~20 frames, driven
inside the render loop, NOT the sim); `freezeTicksLeft > 0` / `slowTicksLeft > 0` →
scene-level color grade (fog/clearColor lerp to icy blue / amber).

`effects.ts` exports:

```ts
export function createEffects(scene: Scene): {
	deathBurst(pos: { x: number; y: number }, color: [number, number, number]): void;
	playerHit(): void; // triggers shake envelope
	update(state: GameState): void; // per-frame: shake decay + status tint
	dispose(): void;
};
```

Death burst: ONE pooled `ParticleSystem` (deep import
`@babylonjs/core/Particles/particleSystem` + required side-effect imports) re-targeted
per burst (`manualEmitCount`), capacity ~200; no per-death allocation. Uses the
particle sprite from `public/game/particle.png` (Task 5 generates it; until then use a
`DynamicTexture` radial dot drawn once at init — replace in Task 5).

Loop diffing: keep a `Map<id, {x,y,color}>` of last-seen enemies (render-side only).

### Task 5: Asset pipeline — seeded generator + committed outputs

**Files:**
- Create: `scripts/gen-assets.ts`
- Create: `public/game/particle.png`, `public/game/ground.png` (generated)
- Modify: `package.json` (script `gen:assets": "node --experimental-strip-types scripts/gen-assets.ts"` — Node ≥ 22.6; verify local node supports strip-types, else use `tsx` via `pnpm dlx`)
- Modify: `src/lib/game/render/effects.ts` (particle sprite from URL), `src/lib/game/render/scene.ts` (ground texture)
- Modify: `docs/game-design.md` (pipeline section + ADR note)

Generator: pure node, zero new runtime deps. Hand-roll minimal PNG encoding (RGBA,
no filter, zlib via `node:zlib` deflateSync, CRC32 table inline — ~60 lines) — this is
the whole pipeline, deterministic from a fixed seed constant:
- `particle.png` 64×64: radial falloff sprite, white core → transparent.
- `ground.png` 512×512: dark base + seeded value-noise cells + faint grid lines
  (tileable: noise sampled on a wrapping lattice).

Script is idempotent (same bytes every run — assert by regenerating in a unit test?
NO — keep tests off binary outputs; instead the script prints sha256 of each output;
document the expected hashes in the script header comment).

Renderer: ground material gets the texture (`new Texture("/game/ground.png", scene)`,
deep import, uScale/vScale ~6); effects particle system swaps to
`new Texture("/game/particle.png", scene)`.

ADR paragraph in `docs/game-design.md`: chose generated-procedural assets over binary
art/glTF (no large binaries, reproducible, reviewable); regeneration = `pnpm gen:assets`.

Snapshot regen rides. Commit: `feat(game): add seeded asset pipeline with generated ground and particle textures`
